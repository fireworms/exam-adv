# 학습 자료 페이지 제작 가이드 — 파트 3 (섹션 7~16)

> **이어서**: 파트 1·2(섹션 1~6)에서 프로젝트 개요·산출물·기술 스택·라우팅·DB 스키마를 확정했다.
> 본 파트는 **공통 페이지·과목별 특화 페이지·데이터 적재·UI·디자인·SM-2·로드맵·체크리스트**를 다룬다.

## 7️⃣ 공통 페이지 사양

공통 페이지는 과목에 관계없이 동일한 컴포넌트를 재사용한다. 과목별 데이터만 달라지며, 인증·북마크·검색·진도 API는 단일 엔드포인트를 공유한다.

---

### 7.1 통합 대시보드 (`/`)

**목적**: 5과목 학습 현황을 한 화면에서 파악하고 다음 학습 항목으로 빠르게 진입.

#### 레이아웃 구성

```
┌─────────────────────────────────────────────────┐
│  시험 D-Day 카운터 (2026-06-20 기준)              │
│  [D-31] 지방직까지 31일 남았습니다               │
├──────────┬──────────┬──────────┬────────────────┤
│  국어    │  영어    │  한국사  │  컴퓨터일반    │
│  열람률  │  열람률  │  열람률  │  열람률        │
│  암기율  │  암기율  │  암기율  │  암기율        │
│  카드 진도│ 카드 진도│ 카드 진도│ 카드 진도      │
├──────────┴──────────┴──────────┴────────────────┤
│  정보보호론 (넓게)  열람률 / 암기율 / 카드 진도  │
├─────────────────────────────────────────────────┤
│  📌 오늘의 복습 카드  (SM-2 next_review_at ≤ 오늘)│
│  [영어 어휘 7개] [정보보호 함정 3개] [컴일 개념 2개]│
├─────────────────────────────────────────────────┤
│  🔖 최근 북마크 (최대 5개)                       │
│  ⏱️ 오늘 학습 시간 요약                          │
└─────────────────────────────────────────────────┘
```

#### 컴포넌트 데이터 소스

| UI 요소 | 데이터 소스 | API |
|---------|------------|-----|
| D-Day 카운터 | 환경변수 `EXAM_DATE=2026-06-20` | 클라이언트 계산 |
| 과목별 열람률 | `user_views` (item_type별 count) | `GET /api/progress/summary` |
| 과목별 암기율 | `vocab_progress`, `cs_concept_progress`, `user_trap_progress` | 위 API 포함 |
| 오늘의 복습 카드 | `next_review_at ≤ NOW()` 쿼리 | `GET /api/review/due` |
| 최근 북마크 | `user_bookmarks ORDER BY created_at DESC LIMIT 5` | `GET /api/bookmarks?limit=5` |

#### 구현 포인트

- 5과목 진도 카드는 `<SubjectProgressCard subject={...} />` 공통 컴포넌트 재사용.
- D-Day가 7 이하이면 카운터를 **빨간색 강조 + 펄스 애니메이션** 처리.
- 오늘의 복습 카드가 0개이면 "오늘 복습 완료 🎉 다음 복습은 내일입니다" 메시지 표시.

### 7.2 과목 허브 (`/[subject]/dashboard`)

**목적**: 과목 진입 시 빈출 TOP 10·진도 요약·함정 패턴을 한 화면에서 확인.

#### 섹션 구성

1. **빈출 TOP 10** — 과목별 JSON(`top_10_areas` / `key_concepts` 상위 10개)을 카드 리스트로 렌더. 각 카드에 "열람" / "암기" 버튼 내장.
2. **과목 진도 요약** — 도넛 차트(Recharts) + 텍스트: 전체 학습 항목 대비 완료 비율.
3. **오늘 복습할 카드** — 해당 과목의 SM-2 만료 카드만 필터.
4. **함정 패턴 바로가기** — `trap_patterns` 상위 3개 미리보기 → "전체 보기" 링크.
5. **트렌드 분석 바로가기** — `트렌드분석.md` 존재 시 링크 표시, 없으면 "준비 중" 배지.

### 7.3 요약 노트 뷰어 (`/[subject]/notes/master`, `/table`, `/json`)

#### 마스터 노트 (`/notes/master`)

- **렌더러**: `react-markdown` + `remark-gfm` (표·코드블록·강조)
- `요약노트_마스터.md` 파일을 `/api/notes/[subject]/master`에서 스트리밍 반환.
- **목차 사이드바**: `##` 헤딩을 파싱해 자동 생성. 클릭 시 스크롤 앵커 이동.
- **PDF 다운로드 버튼**: `요약노트_마스터.pdf`를 Supabase Storage URL로 직접 연결.
- **북마크**: 섹션 단위(`item_type = 'note_chapter'`, `item_key = 'master_##섹션명'`) 저장.
- **뷰 기록**: 섹션 진입 시 `user_views` UPSERT.

#### 표중심 노트 (`/notes/table`)

- `요약노트_표중심.md` 렌더. GFM 테이블이 핵심이므로 수평 스크롤 처리(`overflow-x: auto`).
- 표 번호(1~25)를 좌측 네비게이션 바에 목록 표시.
- **"표 강조 모드"** 토글: 활성화 시 표 외 본문 텍스트를 흐리게(`opacity-40`) 처리하여 표에 집중.

#### JSON 트리 뷰 (`/notes/json`)

- `요약노트_JSON.json`의 최상위 키를 펼침/접힘 형태로 렌더.
- 학습 카드·인터랙션 컴포넌트와 연계: 해당 키 옆에 "카드로 학습" 버튼 표시.
- 검색창: 키·값 텍스트 내 실시간 필터(`filter` + 하이라이트).

### 7.4 함정 패턴 학습 (`/[subject]/traps`)

**목적**: 수험생이 가장 자주 틀리는 함정 TOP 15를 학습·체크.

#### 레이아웃

```
┌────────────────────────────────────────┐
│  함정 TOP 15  [과목 선택 드롭다운]      │
│  진도: ██████░░░░░ 8/15 완료           │
├──────┬─────────────────────────────────┤
│  #1  │  ❌ 잘못된 형태                  │
│  빈도│  ✅ 올바른 형태                  │
│  높음│  💡 회피법                       │
│      │  📌 [이해 완료] 버튼             │
├──────┼─────────────────────────────────┤
│  #2  │  ...                            │
└──────┴─────────────────────────────────┘
```

#### 동작 사양

- **"이해 완료"** 클릭 → `user_trap_progress.understood = true` UPSERT.
- **"관련 기출 보기"** 버튼: `exam_appeared_in` 값이 있을 때만 표시 → 외부 문제풀이 앱 딥링크(15장 참조).
- **SM-2 연동**: 함정도 SM-2 복습 주기에 포함. "이해 완료" 시 초기 `next_review_at = NOW() + 1일` 설정.
- **카테고리 필터**: `trap_category`(숫자·매칭·법령조항·개념) 탭으로 필터.

### 7.5 통합 검색 (`/[subject]/search`, 전역 헤더 검색)

- **PostgreSQL FTS**: `to_tsvector('simple', ...)` 인덱스를 활용한 한글·한자 검색.
- **검색 범위**: 현재 과목 내 어휘·인물·사료·법규·함정·노트 챕터.
- **전역 검색**(헤더): 5과목 전체 대상, 결과를 과목별 그룹으로 표시.
- **결과 카드**: 항목 유형 배지(어휘/인물/법규/함정) + 요약 + 바로가기 링크.
- **딥링크**: 결과 클릭 시 해당 상세 페이지로 이동, 검색어를 URL 쿼리(`?highlight=...`)로 전달하여 페이지 내 하이라이트 처리.

API 예시:
```typescript
// GET /api/search?q=SHA-256&subject=infosec&limit=20
type SearchResult = {
  subject: string;
  item_type: string;   // 'vocab' | 'trap' | 'law_article' | 'figure' | 'crypto' | ...
  item_key: string;
  title: string;
  summary: string;
  href: string;        // 클라이언트 내부 경로
};
```

### 8.1 🇰🇷 국어 특화

#### 8.1.1 영역별 학습 (`/korean/areas`, `/korean/areas/[areaKey]`)

**목록 페이지**

- 각 카드: 순위 배지 · 영역명 · 학습 비중(%) · 난이도 배지 · 회당 출제 빈도.

**상세 페이지 (`/korean/areas/[areaKey]`)**

- 영역 정의·핵심 전략·빈출 유형·예시 지문을 마크다운으로 렌더.
- **체크리스트**: 해당 영역 학습 완료 항목 수동 체크.
- **함정 패턴 연관**: 이 영역과 관련된 함정 카드 3개 노출.

#### 8.1.2 회차 매핑 검색 (`/korean/exam-mapping`)

- `exam_round_mapping` 데이터에서 시험 회차(연도·직렬·번호) 입력 → 해당 문제의 출제 영역·키워드 반환.

- 데이터가 없는 회차는 "데이터 없음" 안내.

#### 8.1.3 시험 직전 체크리스트 (`/korean/checklist`)

### 8.2 🇬🇧 영어 특화

#### 8.2.1 어휘 카드 (`/english/vocabulary`, `/english/vocabulary/[category]`)

**카테고리 목록 페이지**

| 카테고리 키 | 표시명 | 단어 수 |
|------------|--------|--------|
| `environment_climate` | 환경·기후 | — |
| `ai_digital` | AI·디지털 | — |
| `administration_policy` | 행정·정책 | — |
| `health_welfare` | 건강·복지 | — |
| `economy_labor` | 경제·노동 | — |
| `multi_meaning` | 다의어 | — |

**카드 학습 페이지 (`/[category]`)**

- **플립 카드 UI** (Framer Motion):
  - 앞면: 영어 단어 + 발음기호 + TTS 버튼(Web Speech API)
  - 뒷면: 한국어 뜻 + 예문 + 예문 번역 + 동의어/반의어
- **평가 버튼**: `어려움(0)` / `보통(1)` / `쉬움(2)` → SM-2 `quality` 입력.
- **진도 바**: 현재 세션 내 완료 카드 / 전체 카드.
- **기출 배지**: `exam_appeared_in` 값 있으면 칩 형태로 표시 → 클릭 시 딥링크.
- **필터**: SM-2 상태별(`new` / `learning` / `reviewed` / `mastered`) 탭 필터.

#### 8.2.2 관용구 (`/english/idioms`)

- 4개 카테고리(`workplace` / `decision_making` / `emotion` / `time`) 탭.
- 각 관용구 카드: 표현 + 뜻 + 예문 + 직역.
- **매칭 미니 퀴즈**: 카테고리 내 관용구 8개를 뜻과 무작위 매칭 → 드래그 앤 드롭 or 클릭 선택.
  - 정답/오답 즉시 피드백(toast).
  - 결과를 `user_mini_quiz_attempts`에 기록(`quiz_type = 'idiom_matching'`).

#### 8.2.3 문법 패턴 (`/english/grammar-patterns`, `/english/grammar-patterns/[id]`)

- 10개 패턴 목록 → 클릭 시 상세.
- 상세: 패턴 형식(코드블록) + 한국어 의미 + 예문 + 동등 표현 + 핵심 포인트.
- **패턴별 미니 퀴즈**: 해당 패턴이 적용된 문장 3개 제시 → "올바른 변환형 선택" (4지선다가 아닌 빈칸 드롭다운 방식).
  - 결과: `quiz_type = 'grammar_pattern_quiz'`.

#### 8.2.4 실무 영어 (`/english/business-english`)

- 이메일 작성 컨텍스트(`email_opening` / `email_body` / `meeting` / `government`) 탭.
- **이메일 시뮬레이션**: 시나리오 제시 → 빈칸에 알맞은 표현 선택 → 완성된 이메일 미리보기.
- 각 표현 카드는 플립 카드(앞: 한국어 상황, 뒤: 영어 표현).

#### 8.2.5 안내문 함정 퀴즈 (`/english/announcements`)

- 8종 함정 카테고리(`time_day` / `registration` / `cost_discount` / `time_conversion` / `virtual_offline` / `capacity` / `closure` / `location_split`) 카드.
- **미니 퀴즈**: 안내문 지문(학습용 예시 텍스트) 제시 → "다음 중 잘못 이해한 것은?" 형식 (단답·O/X).
  - `quiz_type = 'announcement_trap'`.

### 8.3 📜 한국사 특화

#### 8.3.1 왕조 연표 (`/korean-history/timeline`)

- **vis-timeline** 또는 **D3.js** 수평 연표.
- 시대(선사~현대) → 왕조 → 주요 사건 레이어 구조.
- **인터랙션**:
  - 사건 클릭 → 팝업: 연도·설명·관련 인물 링크.
  - 확대/축소(마우스 휠), 드래그 이동.
  - 카테고리 필터(정치/경제/문화/외교) 토글.
- **모바일**: 수직 타임라인으로 반응형 전환.

데이터 소스: `historical_events` + `eras` + `dynasties` JOIN.

#### 8.3.2 인물 사전 (`/korean-history/historical-figures`, `/[id]`)

**목록**

- `figures` 테이블 전체 목록. 시대별 필터 + 역할별 필터(`scholar`/`reformer`/`activist`/`general`).
- 검색: 이름(한글·한자)·호·자·시호.
- 정렬: 중요도 DESC (기본) / 시대순.

**상세 (`/[id]`)**

```
┌──────────────────────────────────────────┐
│  [인물 이미지 또는 아이콘]                │
│  이름 / 한자명 / 호 / 자 / 시호          │
│  생몰년 · 역할                            │
│  소속 왕조/시대 (링크)                    │
├──────────────────────────────────────────┤
│  📖 요약                                  │
│  🏆 주요 업적 (achievements JSONB 리스트) │
├──────────────────────────────────────────┤
│  🕸️ 인물 관계도 (react-force-graph)      │
│    스승-제자 / 동지 / 적대 / 가족        │
├──────────────────────────────────────────┤
│  📜 관련 사료 (historical_documents 링크) │
│  📅 관련 사건 (historical_events 링크)   │
│  🎯 기출 출제 횟수                        │
```

### 9.3 기출 메타데이터 추출 (`inject_exam_meta.ts`)

```typescript
// set*.json의 각 문제 explanation 필드에서 추출
// 예: "[관련 기출: 2025 지방직 1번 / 핵심 개념: SHA-256 출력 크기]"
const META_REGEX = /\[관련 기출:\s*([^\]\/]+?)(?:\s*\/\s*핵심 개념:\s*([^\]]+))?\]/g;

async function injectExamMeta(setFiles: string[]) {
  for (const file of setFiles) {
    const data = JSON.parse(await fs.readFile(file, 'utf-8'));
    for (const q of data.questions) {
      const matches = [...(q.explanation ?? '').matchAll(META_REGEX)];
      for (const m of matches) {
        const examRef = m[1].trim();   // "2025 지방직 1번"
        const conceptKey = m[2]?.trim(); // "SHA-256 출력 크기"
        // vocab_words / cs_concepts / law_articles 등 해당 레코드의
        // exam_appeared_in 컬럼 UPSERT
        await upsertExamMeta(examRef, conceptKey);
      }
    }
  }
}
```

## 🧮 12️⃣ 학습 진도 추적 (SuperMemo SM-2)

### 12.1 SM-2 알고리즘 구현

```typescript
// lib/sm2.ts

/** 품질 점수 (사용자 입력) */
type Quality = 0 | 1 | 2 | 3 | 4 | 5;

/** 카드별 SM-2 상태 */
interface SM2State {
  repetitions: number;    // 연속 성공 횟수
  easeFactor: number;     // 난이도 계수 (초기 2.5)
  interval: number;       // 다음 복습까지 일수
}

/**
 * SM-2 계산
 * @param state 현재 카드 상태
 * @param quality 사용자 평가 (0~5)
 * @returns 업데이트된 SM-2 상태
 */
export function sm2Update(state: SM2State, quality: Quality): SM2State {
  const { repetitions, easeFactor, interval } = state;

  if (quality < 3) {
    // 실패: 처음부터
    return { repetitions: 0, easeFactor, interval: 1 };
  }

  const newEF = Math.max(
    1.3,
    easeFactor + 0.1 - (5 - quality) * (0.08 + (5 - quality) * 0.02)
  );

  let newInterval: number;
  if (repetitions === 0) newInterval = 1;
  else if (repetitions === 1) newInterval = 6;
  else newInterval = Math.round(interval * newEF);

  return {
    repetitions: repetitions + 1,
    easeFactor: newEF,
    interval: newInterval,
  };
}

/** quality 매핑 테이블 (UI 버튼 → SM-2 quality) */
export const QUALITY_MAP = {
  // 어휘·개념 카드
  어려움: 2 as Quality,
  보통:   3 as Quality,
  쉬움:   5 as Quality,
  // 컴퓨터일반 플래시카드
  헷갈림: 2 as Quality,
  // '보통'은 위와 동일 (3)
  익숙:   5 as Quality,
  // 함정 카드 (이해 완료 = 성공)
  이해완료: 4 as Quality,
  // 미니 퀴즈
  정답: 4 as Quality,
  오답: 1 as Quality,
};
```

### 12.2 SM-2 복습 API

```typescript
// app/api/review/rate/route.ts (POST)
// Body: { table: 'vocab_progress'|'cs_concept_progress'|'user_trap_progress', id: number, quality: Quality }

export async function POST(req: Request) {
  const { table, id, quality } = await req.json();
  const user = await getUser(req);

  // 현재 SM-2 상태 조회
  const { data: row } = await supabase
    .from(table).select('repetitions, ease_factor, interval').eq('id', id).single();

  // SM-2 계산
  const next = sm2Update(
    { repetitions: row.repetitions, easeFactor: row.ease_factor, interval: row.interval },
    quality
  );

  // 다음 복습일 계산
  const nextReviewAt = addDays(new Date(), next.interval);

  // 상태 업데이트
  await supabase.from(table).update({
    repetitions: next.repetitions,
    ease_factor: next.easeFactor,
    interval: next.interval,
    next_review_at: nextReviewAt.toISOString(),
    last_reviewed_at: new Date().toISOString(),
    mastery_level: Math.min(5, next.repetitions),
    status: next.repetitions >= 5 ? 'mastered' : next.repetitions >= 2 ? 'reviewed' : 'learning',
  }).eq('id', id);

  return Response.json({ nextReviewAt, interval: next.interval });
}
```

## 🔗 15️⃣ 외부 문제풀이 앱과의 연계

### 15.1 딥링크 설계

외부 문제풀이 앱(set1~3 기반 4지선다 앱)과의 연계는 **URL 딥링크**로 구현한다.

```typescript
// lib/deeplink.ts

const QUIZ_APP_BASE_URL = process.env.NEXT_PUBLIC_QUIZ_APP_URL ?? '';

/**
 * 기출 메타데이터 → 외부 문제풀이 앱 딥링크 URL 생성
 * exam_appeared_in 예: "2025 지방직 1번"
 */
export function buildQuizDeepLink(examAppearedIn: string): string | null {
  if (!QUIZ_APP_BASE_URL || !examAppearedIn) return null;

  // "2025 지방직 1번" → { year: 2025, type: '지방직', no: 1 }
  const match = examAppearedIn.match(/(\d{4})\s*(지방직|국가직|서울시|법원직)\s*(\d+)번/);
  if (!match) return null;

  const [, year, examType, no] = match;
  const params = new URLSearchParams({ year, examType, questionNo: no });
  return `${QUIZ_APP_BASE_URL}/practice?${params.toString()}`;
}
```

### 15.2 딥링크 표시 규칙

- `exam_appeared_in` 값이 있는 카드·법규 조문·인물 상세에 `<ExamMetaBadge>` 표시.
- `QUIZ_APP_BASE_URL` 환경변수가 비어 있으면 배지는 표시되되 링크 비활성화("외부 앱 미연결" 툴팁).
- 딥링크 URL은 `target="_blank" rel="noopener noreferrer"` 으로 새 탭 열기.

### 15.3 향후 연계 확장 옵션

| 방향 | 구현 방법 |
|------|---------|
| 사료 → 관련 기출 문제 | 사료 상세 페이지 하단 "관련 기출 문제 풀어보기" 버튼 |
| 암호 실습실 → 관련 문제 | 알고리즘 카드 내 ExamMetaBadge |
| 통합 검색 결과 → 문제 풀이 | 검색 결과 카드에 "기출 문제 보기" 아이콘 버튼 |
| 역방향(문제풀이 앱 → 학습 앱) | 문제풀이 앱이 `학습앱URL/[subject]/traps?highlight=SHA-256` 형태로 딥링크 |
