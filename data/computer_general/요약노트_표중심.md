## 표 1. 수의 음수 표현 방식 비교

| 방식 | 표현 범위 (n비트) | 0의 표현 | 특징 |
|------|-----------------|---------|------|
| 부호-크기 | -(2ⁿ⁻¹-1) ~ +(2ⁿ⁻¹-1) | 2개 (+0, -0) | 직관적이나 0 두 개 |
| 1의 보수 | -(2ⁿ⁻¹-1) ~ +(2ⁿ⁻¹-1) | 2개 (+0, -0) | 비트 반전, 0 두 개 |
| **2의 보수** | **-2ⁿ⁻¹ ~ +(2ⁿ⁻¹-1)** | **1개** | **현 컴퓨터 표준** |

**계산**: 절댓값 → 1의 보수(비트 반전) → +1 = 2의 보수

## 표 6. 페이지 교체 알고리즘

| 알고리즘 | 동작 | 특징 |
|---------|------|------|
| **FIFO** | 가장 먼저 들어온 페이지 교체 | Belady 이상 현상 발생 |
| **LRU** | 가장 오래 안 쓴 페이지 교체 | 스택 알고리즘 (Belady 없음) |
| **LFU** | 사용 빈도 최저 페이지 교체 | 초기 페이지 불리 |
| **MFU** | 가장 자주 쓴 페이지 교체 | LFU의 반대 |
| **Optimal** | 미래 가장 안 쓸 페이지 교체 | 이론적 최저 폴트 (구현 불가) |
| **2nd Chance** | FIFO + 참조 비트 검사 | 클럭 알고리즘 |
| **NUR** | 참조/수정 비트 4클래스 | 효율적 LRU 근사 |

## 표 8. RAID 레벨

| 레벨 | 방식 | 최소 디스크 | 장애 허용 | 사용 효율 |
|------|------|----------|----------|----------|
| **RAID 0** | 스트라이핑 | 2 | 0개 | 100% |
| **RAID 1** | 미러링 | 2 | 1개 | 50% |
| RAID 2 | 해밍 코드 | 3 | 1개 | 가변 |
| RAID 3 | 바이트 단위 + 패리티 | 3 | 1개 | (n-1)/n |
| RAID 4 | 블록 단위 + 전용 패리티 | 3 | 1개 | (n-1)/n |
| **RAID 5** | 블록 단위 + **분산 패리티** | 3 | 1개 | (n-1)/n |
| **RAID 6** | 블록 단위 + **이중 패리티** | 4 | **2개** | (n-2)/n |
| **RAID 10** | 미러링 후 스트라이핑 | 4 | 다수 | 50% |

⚠️ **함정**: RAID 6는 RAID 5보다 **패리티 더 많고 더 느림**

## 표 9. 정규화 단계

| 정규형 | 만족 조건 | 제거 대상 |
|--------|---------|----------|
| **1NF** | 모든 속성이 원자 값 | 다치 속성, 그룹 속성 |
| **2NF** | 1NF + 완전 함수 종속 | **부분 함수 종속** |
| **3NF** | 2NF + 비주요속성 → 비주요속성 없음 | **이행 함수 종속** |
| **BCNF** | 모든 결정자가 슈퍼키 | (3NF 강화) |
| **4NF** | BCNF + 다치 종속 없음 | 다치 종속 |
| **5NF** | 4NF + 조인 종속 없음 | 조인 종속 |

⚠️ BCNF는 3NF를 포함 (BCNF ⊃ 3NF)

## 표 11. 트랜잭션 고립 수준 (Isolation Level)

| 수준 | Dirty Read | Non-repeatable Read | Phantom Read |
|------|-----------|-------------------|------------|
| READ UNCOMMITTED | ⭕ 발생 | ⭕ 발생 | ⭕ 발생 |
| READ COMMITTED | ❌ 방지 | ⭕ 발생 | ⭕ 발생 |
| REPEATABLE READ | ❌ 방지 | ❌ 방지 | ⭕ 발생 |
| **SERIALIZABLE** | ❌ 방지 | ❌ 방지 | ❌ 방지 |

- **Dirty Read**: 커밋되지 않은 데이터 읽음
- **Non-repeatable Read**: 같은 쿼리 반복 시 값 변경됨
- **Phantom Read**: 같은 쿼리 반복 시 행 추가/삭제 발견

## 표 15. TCP vs UDP

| 구분 | TCP | UDP |
|------|-----|-----|
| 연결 | **연결 지향** (3-way handshake) | **비연결** |
| 신뢰성 | 보장 (재전송, ACK) | 보장 안 함 |
| 순서 | 보장 | 보장 안 함 |
| 흐름 제어 | 있음 (윈도우) | 없음 |
| 혼잡 제어 | 있음 | 없음 |
| **헤더 크기** | **20-60바이트** | **8바이트 (고정)** |
| 속도 | 느림 | 빠름 |
| 용도 | HTTP, FTP, SMTP, Telnet | DNS, DHCP, SNMP, VoIP, 게임, 스트리밍 |

## 표 16. 정렬 알고리즘

| 알고리즘 | 평균 | 최악 | 최선 | 공간 | 안정성 |
|---------|------|------|------|------|--------|

| **퀵 정렬** | **O(n log n)** | O(n²) | O(n log n) | O(log n) | ❌ |

## 표 17. BFS vs DFS

| 구분 | BFS (너비 우선) | DFS (깊이 우선) |
|------|---------------|---------------|
| 자료구조 | **큐 (Queue)** | **스택 (Stack) or 재귀** |
| 탐색 순서 | 레벨 순서 | 깊이 끝까지 |
| 최단 경로 | 가중치 없는 그래프에서 보장 | 보장 안 됨 |
| 메모리 | 많이 사용 (큐 폭) | 적게 사용 (깊이) |
| 시간 복잡도 | O(V+E) | O(V+E) |
| 활용 | 최단 경로, 레벨 탐색 | 사이클 검사, 위상 정렬, 백트래킹 |

## 표 18. 해시 충돌 해결

| 방법 | 동작 | 특징 |
|------|------|------|
| **선형 조사 (Linear Probing)** | (h+i) mod m | 1차 군집화 발생 |
| **이차 조사 (Quadratic)** | (h+i²) mod m | 2차 군집화 |
| **이중 해싱 (Double Hashing)** | (h₁+i·h₂) mod m | 군집화 최소 |
| **체이닝 (Chaining)** | 연결 리스트로 충돌 키 저장 | 적재율 > 1 가능 |

## 표 22. 테스트 기법

| 분류 | 기법 | 설명 |
|------|------|------|
| **화이트박스** (코드 기반) | 기본 경로 (Basis Path) | 순환 복잡도 활용 |
| | 문장 커버리지 | 모든 문장 실행 |
| | 분기 커버리지 | 모든 분기 (참/거짓) |
| | 조건 커버리지 | 모든 조건의 참/거짓 |
| | 경로 커버리지 | 모든 가능 경로 |
| **블랙박스** (명세 기반) | 동등 분할 | 유효/무효 구간 |
| | **경계값 분석** | 경계 ±1 |
| | 원인-결과 그래프 | 인과 관계 |
| | 비교 테스트 | 다른 SW와 비교 |
| | 오류 추측 | 경험 기반 |

⚠️ **함정**: 경계값 분석은 **블랙박스** (화이트박스 아님)

### 테스트 단계 V-모델
```
요구분석 ↔ 인수 테스트
   ↓             ↑
설계   ↔   시스템 테스트
   ↓             ↑
상세설계 ↔ 통합 테스트
   ↓             ↑
구현     →   단위 테스트
```
