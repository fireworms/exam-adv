> **목적**: 9급 공무원 시험 대비 **5과목 통합 학습 자료 웹앱** 제작 가이드.
> Claude Code, Cursor 등 코딩 에이전트나 개발자에게 작업 의뢰 시 본 문서를 산출물과 함께 전달.
>
> **대상 과목**: 국어 · 영어 · 한국사 · 컴퓨터일반 · 정보보호론
> **타깃 사용자**: 9급 공무원 수험생 (시험 D-30 ~ D-1 집중 사용)
> **시험 일자 예시**: 2026년 6월 20일 (지방직)

## 🎯 본 가이드의 범위

본 가이드가 다루는 것은 **학습 자료 페이지**다. **세트(set1/2/3) 기반 4지선다형 문제풀이는 별도 앱으로 이미 구현되어 있으므로 본 통합 앱의 범위에서 제외한다.**

| 구분 | 본 가이드 범위 | 별도 앱 (이미 존재) |
|------|---------------|-------------------|
| ✅ **본 통합 학습 자료 앱** | 요약노트 뷰어, 어휘 카드(간격 반복), 인물 사전, 사료 모음, 역사 지도, 시대 비교, 암호 실습실, 법규 탐색기, ISMS-P 트리, 접근통제 시뮬레이터, 함정 학습, 플래시카드, 코드 트레이싱, 회차 매핑 검색, 학습 진도·체크리스트, 매칭/단답 미니 퀴즈 등 **학습 보조 인터랙션** | — |
| ❌ **범위 외** | (제외) | `set1.json`/`set2.json`/`set3.json` 기반의 4지선다형 문제 풀이, 오답 노트, 모의고사 시뮬레이션 모드, 4지선다 자동 채점 |

> 📌 **보존 원칙**: 4지선다형 set 풀이를 제외한 모든 학습 보조 인터랙션은 유지한다. 어휘 카드의 "어려움/보통/쉬움" 평가, 관용구 매칭 미니 퀴즈, 문법 패턴별 미니 퀴즈, 컴퓨터일반 코드 트레이싱, 정보보호론 RSA 계산기·BLP 시뮬레이션 등 학습 보조 기능은 모두 유지된다.

**`set1.json` / `set2.json` / `set3.json`** 은 학습 자료 페이지에서 **직접 풀이용으로 적재하지 않는다.** 대신 다음 용도로만 *참조*한다:
- 어휘·문법 패턴·인물·사료·암호·법규 카드 등의 "기출 출제" 메타데이터 추출 (예: "이 어휘는 2025 지방직 1번에 출제됨")
- 함정 카드의 예시 문구 추출
- 향후 문제풀이 앱과의 **딥링크 연계 옵션**(예: 사료 페이지 → "관련 기출 문제 풀어보기" 버튼 클릭 시 외부 문제풀이 앱의 해당 문제로 이동)

상충 지점은 모두 **정보보호론 가이드(최신본)** 기준으로 통합했으며, 국어 가이드의 FastAPI+SQLite+React+Vite 스택은 채택하지 않고 Next.js 15+TypeScript+Supabase로 통일한다.

## 1️⃣ 프로젝트 개요

### 1.1 목적

수험생이 5과목(국어·영어·한국사·컴퓨터일반·정보보호론) **학습 자료**를 하나의 통합 웹앱에서 열람·학습·암기할 수 있도록 한다. 과목별 특성에 맞춘 특화 학습 페이지를 제공하되, 공통 인프라(인증·열람 진도·북마크·검색·통계)는 단일 시스템으로 운영한다.

실제 4지선다형 문제 풀이·채점은 별도 앱에서 처리하므로, 본 앱은 **이론·암기·개념 학습·학습 보조 인터랙션**에 집중한다.

### 1.2 핵심 가치 제안

| 가치 | 구현 방식 |
|------|----------|
| **통합 진도 관리** | 5과목의 노트 열람·암기 카드·미니 퀴즈 진행률을 단일 사용자 계정으로 추적 |
| **과목별 특화 학습** | 한국사 연표·인물 관계도, 정보보호 암호실습실·법규 탐색기, 영어 어휘 카드·실무 영어 등 |
| **공통 인프라 재사용** | 인증, 학습 진도, 북마크, 통합 검색은 단일 시스템 |
| **간격 반복 학습** | SuperMemo SM-2 알고리즘으로 어휘·개념·함정 망각 곡선 대응 |
| **시험 직전 모드** | D-7부터 당일까지 일자별 학습 플로우 자동 제공 |

### 1.3 사용자 시나리오

- **수험생 A**: 시험까지 3개월 → 매일 영어 어휘 30개 + 한국사 인물 5명 + 정보보호 함정 10개 학습
- **수험생 B**: 외부 모의고사 앱 점수 확인 후 약점 영역 노트 정독 + 미니 퀴즈로 보강
- **수험생 C**: 시험 1주 전 → 5과목 표중심 노트 + 함정 TOP 15 + 직전 체크리스트 집중

## 2️⃣ 산출물 구성

### 2.1 표준 산출물 (과목당 9개)

각 과목은 다음 9개 파일로 구성된 패키지를 갖는다(정보보호론·컴퓨터일반 기준).

| 파일명 | 형식 | 본 학습 자료 앱에서의 용도 |
|--------|------|------|
| `set1.json` | JSON | **참조용**: 기출 출제 메타데이터 추출 (DB 적재 ❌) |
| `set2.json` | JSON | 동일 |
| `set3.json` | JSON | 동일 |
| `요약노트_마스터.md` | Markdown | 마스터 노트 뷰어 (24+ 페이지) |
| `요약노트_마스터.pdf` | PDF | 마스터 노트 PDF 뷰어 |
| `요약노트_표중심.md` | Markdown | 표 중심 노트 뷰어 (25개 표) |
| `요약노트_표중심.pdf` | PDF | 표중심 PDF 뷰어 |
| `요약노트_JSON.json` | JSON | **메인 데이터 소스** (DB 적재) |
| `트렌드분석.md` | Markdown | 트렌드 분석 뷰어 (회차별 빈도·함정) |

### 2.2 과목별 산출물 현황 (2026년 6월 시점)

| 과목 | set1~3 | 마스터 노트 | 표중심 | 요약노트 JSON | 트렌드 분석 | 비고 |
|------|:----:|:------:|:----:|:----:|:------:|------|
| 국어 | ✅ | ✅ | ✅ | ✅ | ⚠️ 미작성 | 8개 파일 |

## 3️⃣ 권장 기술 스택

### 3.1 코어 스택

- **Frontend**: Next.js 15 (App Router) + TypeScript + Tailwind CSS
- **UI 컴포넌트**: shadcn/ui (Radix UI + Tailwind 기반)
- **Backend**: Next.js API Routes (또는 분리형 Express/FastAPI)
- **Database**: Supabase (PostgreSQL + Auth + Storage)
- **상태관리**: Zustand 또는 React Query (서버 상태)

### 3.2 시각화·렌더링 라이브러리

| 용도 | 라이브러리 |
|------|----------|
| 마크다운 렌더 | `react-markdown` + `remark-gfm` (표 지원) |
| 수식 렌더 | KaTeX (RSA·디피헬만·해시 수식) |
| 코드 하이라이트 | `prism-react-renderer` 또는 Prism.js / highlight.js |
| 차트 | Recharts (열람 진도, 마스터리 시각화) |
| 지도 | Leaflet.js + react-leaflet (한국사 역사 지도) |
| 타임라인 | vis-timeline 또는 D3.js (한국사 연표) |
| 관계도 | react-force-graph 또는 vis-network (한국사 인물 관계도) |
| 카드 애니메이션 | Framer Motion (영어 어휘 플립 카드) |
| PDF 뷰어 | react-pdf (요약 노트 PDF, lazy load) |
| PDF 생성 | jsPDF 또는 react-pdf (시대 비교 내보내기) |
| 아이콘 | lucide-react |
| 토스트 | react-hot-toast (정답/오답 피드백, 학습 완료 알림) |

### 3.3 음성·접근성

- **TTS**: Web Speech API (브라우저 내장, 별도 키 불필요)
  - 영어 어휘 발음: `utterance.lang = 'en-US'`
  - 한국사 사료 현대어 번역 읽기: `utterance.lang = 'ko-KR'`
  - 정보보호론: TTS 불필요 (수식 음성은 KaTeX AccessibleMathML로 선택적 지원)

### 3.4 검색·인증

- **검색**: PostgreSQL Full-Text Search (한글·한자 통합 검색)
- **인증**: Supabase Auth (이메일/소셜 로그인)

### 3.5 배포

- **호스팅**: Vercel (Next.js 최적화) 또는 자체 서버
- **DB 호스팅**: Supabase 또는 Neon
- **이미지/사료/오디오 저장**: Supabase Storage 또는 Cloudflare R2

## 4️⃣ 사이트 구조 및 라우팅

├── /subjects                  # 과목 선택 페이지
│
├── /[subject]/                # 과목 허브 (subject = korean | english | korean-history | computer-general | infosec)
│   ├── /dashboard             # 과목 대시보드 (빈출 TOP 10, 진도 요약)
│   ├── /notes                 # 요약 노트 뷰어
│   │   ├── /master            # 마스터 노트
│   │   ├── /table             # 표중심 노트
│   │   └── /json              # JSON 데이터 트리 뷰
│   ├── /trends                # 트렌드 분석 (회차별 빈도)

> 📌 **외부 문제풀이 앱 연계**: 학습 항목 화면(예: 사료 상세, 어휘 카드)의 "관련 기출" 칩은 클릭 시 외부 문제풀이 앱의 해당 문제로 딥링크 이동한다(15장 참조).

### 4.2 공통 레이아웃

- **상단 헤더**: 사이트 타이틀, 과목 전환 드롭다운, 통합 검색창, 다크모드 토글, 프로필 메뉴
- **좌측 사이드바**(데스크탑) / **상단 햄버거 메뉴**(모바일): 현재 과목 내비게이션
- **하단 푸터**: 학습 슬로건(국어 `final_motto` 등), 시험 정보, 산출물 메타 (`analysis_scope`)

## 5️⃣ DB 스키마

학습 자료 페이지는 4지선다형 문제풀이 데이터를 직접 적재하지 않으므로 `questions`/`choices`/`user_attempts` 테이블은 두지 않는다. 대신 **학습 콘텐츠 테이블**(어휘/인물/사료/암호/법규/함정 등)과 **사용자 학습 상태 테이블**(열람·암기 진도·북마크·체크리스트·미니 퀴즈 진행)로 구성한다.

### 5.1 공통 사용자 상태 테이블 (5과목 공유)
